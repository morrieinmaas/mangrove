# v0.3.0 Plan 2 — `mangrove fmt` Formatter Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A `mangrove fmt` formatter that produces a canonical, deterministic layout of a
`.mang` file by walking the lossless CST — preserving comments and meaning, never changing the
document's content hash.

**Architecture:** A new `mangrove-fmt` crate walks the CST token stream (`parse_cst(src).syntax()`)
in document order, re-emitting each token with normalized surrounding whitespace and depth-based
indentation. It is a **whitespace/indentation normalizer**, not a line-reflow engine: it preserves
the author's line-break structure (an inline record stays inline; a multi-line one stays
multi-line), normalizes spacing/indent within, drops trailing commas, collapses blank-line runs,
and keeps comments attached to their line. Line-reflow (auto-wrapping long lines, expanding
records) is explicitly out of scope for v1.

**Tech Stack:** Rust 2024, the `mangrove-syntax` CST (`rowan`), the existing `mangrove-cli`.

## Global Constraints

- Rust 2024 edition (≥ 1.85); workspace `unsafe_code = "forbid"` — NO `unsafe`.
- Every commit gated on `just ci` (fmt-check → clippy `-D warnings` → build → test). NEVER commit
  unless `just ci` just passed. (`rtk clippy` hook misreports — verify with `rtk proxy cargo clippy`.)
- Conventional Commits; NO `Co-Authored-By` / AI-attribution lines. Trunk-based: commit to `main`.
- **The formatter MUST NOT change meaning.** For evaluable input, `hash(parse(fmt(src))) ==
  hash(parse(src))`. For non-evaluable input, `fmt(src)` re-parses to a CST with identical
  non-trivia tokens. These are hard correctness gates, tested per-task and over the corpus.
- **Idempotence:** `fmt(fmt(x)) == fmt(x)`.
- The formatter consumes only the CST's PUBLIC API: `mangrove_syntax::cst::{parse_cst, Parse,
  SyntaxNode, SyntaxToken, SyntaxKind, SyntaxElement}`. `Parse { green, errors }`,
  `Parse::syntax() -> SyntaxNode`.
- **Node kinds the tree actually contains** (everything else is flat token runs — DO NOT write
  visitors expecting `TYPE_*`/`MATCH_ARM`/`INTERP`/`INTERP_STR` nodes; they are never emitted):
  `DOCUMENT, USE_DECL, TYPE_DEF, UNIT_DEF, PARAM_DECL, FN_DEF, SCHEMA_DECL, BINDING, RECORD,
  FIELD, LIST, SPREAD, LIST_OP_ITEM, REF, CALL, MATCH_EXPR, UNSET, ERROR`. Trivia token kinds:
  `WHITESPACE, COMMENT, NEWLINE, DOC, DIRECTIVE`.

## Canonical style (v1 — the exact rules this formatter enforces)

1. **Indentation:** 2 spaces per nesting level. Nesting increases inside `{ }` and `[ ]` and
   `( )`. A token on its own line is indented by `2 * depth_at_line_start`.
2. **Inline spacing (token adjacency):**
   - exactly one space after `:` and `,`; NO space before `:` or `,`.
   - exactly one space around binary operators `& | = == != >= <= > < && || += =~` and `*`
     (default marker) when between operands.
   - `{`/`[`/`(` followed by content: one space after `{`/`[` for non-empty (`{ a: 1 }`), none for
     `(` (call args: `webapp(env: "prod")`). One space before `}`/`]` for non-empty; `{}`/`[]`
     stay empty (no inner space).
   - `...` (spread) directly precedes its alias: `...base` (no space).
   - `@` annotations: `@key(name)`, `@message("…")` — no space after `@`, no space before `(`.
3. **Line structure (preserve author intent — no reflow):**
   - Preserve the user's newlines between bindings/fields/elements. Do not merge lines or wrap.
   - Within a brace/bracket block, if the original had a newline after `{`/`[`, keep the
     multi-line shape (one item per line); if it was inline, keep inline.
   - Drop trailing commas/newlines immediately before a closing `}`/`]`.
   - Collapse 2+ consecutive blank lines to a single blank line. Trim trailing whitespace on every
     line. Ensure the file ends with exactly one trailing newline.
4. **Comments:** preserve every `#`/`##`/`#!` comment. A comment on its own line is re-indented to
   the current depth. A trailing comment (after code on the same line) stays trailing with one
   space before `#`. Comment TEXT is never modified.
5. **Types & match interiors:** these are flat token runs in the CST. Apply rules 1–2 (spacing)
   to their tokens; do not attempt structural reformatting.

These rules are deterministic and only touch whitespace/comments → meaning is preserved by
construction, and idempotence holds because a second pass finds nothing to change.

```rust
//! `mangrove fmt` — a deterministic, comment-preserving, meaning-preserving
//! formatter that walks the lossless CST and normalizes whitespace/indentation.

use mangrove_syntax::cst::{parse_cst, SyntaxKind};
use mangrove_syntax::ParseError;
use rowan::NodeOrToken;

pub struct FormatResult {
    pub text: String,
    pub errors: Vec<ParseError>,
}

/// Format `src`. Always returns formatted text; `errors` is non-empty if the
/// source had parse errors (the formatter still produces best-effort output,
/// since the CST is lossless even on error).
pub fn format_str(src: &str) -> FormatResult {
    let parse = parse_cst(src);
    let mut out = String::new();
    // v1 identity: emit every token's text in document order.
    for elem in parse.syntax().descendants_with_tokens() {
        if let NodeOrToken::Token(t) = elem {
            out.push_str(t.text());
        }
    }
    FormatResult { text: out, errors: parse.errors }
}
```

> `NodeOrToken` import by matching `elem.as_token()`. Prefer adding `rowan = { workspace = true }`.

```rust
        let once = format_str(src).text;
        let twice = format_str(&once).text;

    /// for non-evaluable input the formatted source re-parses to the same
    /// non-trivia token sequence (structural equivalence).
    fn assert_meaning_preserved(src: &str) {
        let formatted = format_str(src).text;
        match (mangrove_syntax::parse(src), mangrove_syntax::parse(&formatted)) {
            (Ok(a), Ok(b)) => assert_eq!(

            }
        }
```

- [ ] **Step 3: Implement the token-driven emitter.** Replace `format_str`'s body: iterate
  tokens in document order; track the previous non-trivia, non-newline token kind; for each
  significant token decide the separator with `space_between`; emit WHITESPACE tokens are
  DROPPED (regenerated), NEWLINE/COMMENT/DOC/DIRECTIVE tokens pass through (indent handled in

```rust
use mangrove_syntax::cst::SyntaxKind as K;

// Returns the separator string to emit BEFORE `next`, given the previous
// significant token `prev` (None at line/document start). Newlines/indent are
// handled by the caller, not here.
fn space_between(prev: Option<K>, next: K) -> &'static str {
    use K::*;
    let Some(prev) = prev else { return "" };
    // no space before these:
    if matches!(next, COLON | COMMA | R_BRACE | R_BRACKET | R_PAREN | L_PAREN | QUESTION) {
        // exceptions: `}` and `]` DO get a space before them when the block is
        // non-empty — handled by the caller which knows emptiness; here treat
        // R_BRACE/R_BRACKET as "no space" and let the caller override. Keep
        // L_PAREN tight (call args), QUESTION tight (optional field `x?`).
        return "";
    }
    // no space AFTER these (tight prefix):
    if matches!(prev, L_PAREN | L_BRACKET | L_BRACE | AT | DOT_DOT_DOT | BANG) {
        return "";
    }
    // everything else: single space (after `:` `,`, around operators, between
    // a value and an annotation, etc.)
    " "
}
```

> The empty-block (`{}` / `[]`) and non-empty-block (`{ … }` / `[ … ]`) inner-space rule needs the
> caller to know whether `{`/`[` is immediately followed by its closer. Handle it in the caller:
> when emitting `{`/`[`, peek the next significant token; if it's the matching closer, emit no
> inner space; else emit one space. Symmetrically before `}`/`]`. Implement this in `format_str`,
> not in `space_between`. Add the empty-vs-nonempty cases to the test (already in Step 1).

```rust
fn cmd_fmt(args: &[String]) -> std::process::ExitCode {
    use std::process::ExitCode;
    match args.first().map(String::as_str) {
        Some("-") => {
            let mut src = String::new();
            std::io::Read::read_to_string(&mut std::io::stdin(), &mut src).expect("read stdin");
            print!("{}", mangrove_fmt::format_str(&src).text);
            ExitCode::SUCCESS
        }
        Some("--check") => {
            let mut changed = false;
            for path in &args[1..] {
                let src = std::fs::read_to_string(path).unwrap_or_else(|e| { eprintln!("{path}: {e}"); std::process::exit(2) });
                if mangrove_fmt::format_str(&src).text != src { eprintln!("{path}"); changed = true; }
            }
            if changed { ExitCode::from(1) } else { ExitCode::SUCCESS }
        }
        Some(_) => {
            for path in args {
                let src = std::fs::read_to_string(path).unwrap_or_else(|e| { eprintln!("{path}: {e}"); std::process::exit(2) });
                let out = mangrove_fmt::format_str(&src).text;
                if out != src { std::fs::write(path, out).unwrap_or_else(|e| { eprintln!("{path}: {e}"); std::process::exit(2) }); }
            }
            ExitCode::SUCCESS
        }
        None => { eprintln!("usage: mangrove fmt <file…> | --check <file…> | -"); ExitCode::from(2) }
    }
}
```
