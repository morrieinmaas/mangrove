# Mangrove v0.3.0 — CST, Formatter, LSP (Design)

**Date:** 2026-06-25
**Status:** approved design, pre-implementation
**Goal:** Ship a lossless concrete-syntax tree, a `mangrove fmt` formatter, and a full
LSP server (Neovim + Zed), plus a Kubernetes YAML→Mangrove migration guide — without
changing the language's canonical-form / content-addressing guarantees.

## Why

v0.2.0's honest gaps are "no LSP, no formatter." Both require something the current
front end lacks: the lexer **discards ordinary `#` comments** and the AST carries **no
byte spans, no trivia**. The AST is evaluation-only. So a formatter (must preserve
comments) and a full LSP (must map cursor → node, classify tokens, recover from errors)
both need a real syntax foundation first. v0.3.0 builds that foundation once (a lossless
CST) and derives everything from it.

## Non-goals / invariants preserved

- **No change to canonical form, CBOR encoding, or hashing.** Equal values still hash equal.
- **No change to the language surface or semantics.** This is tooling + one example fix.
- **The LSP is strictly read-only and never touches the network.** Diagnostics resolve
  imports only against the *existing* lockfile; the editor never fetches or runs `update`.
  Fail-closed.

## Architecture

Three new crates, each one job. The existing language crates are untouched downstream.

- **`mangrove-cst`** — lossless syntax tree (`rowan`), the event-emitting parser, and
  lowering CST → existing `Document`/`Value` AST.
- **`mangrove-fmt`** — CST → formatted source string.
- **`mangrove-lsp`** — the `lsp-server`-based server; driven by a new `mangrove lsp`
  CLI subcommand over stdio.

The public `parse(src) -> Document` API keeps its signature; internally it routes
`src → CST → lower → Document`. `mangrove-typed`, `-compose`, `-cli`, `-convert`, etc.
do not change — the whole language keeps working through the new front end.

### Dependency choices

- **`rowan`** for the CST: proven (rust-analyzer), lossless green/red trees, a single
  `SyntaxKind` space for tokens + nodes.
- **`lsp-server`** (sync, thread-based — *not* `tower-lsp`) for the LSP: no tokio/async
  runtime, small dependency surface, matches this codebase's sync, lean, `unsafe`-forbidden
  style. The work (parse + type/compose analysis) is synchronous anyway.

## Component 1 — Lossless CST (`mangrove-cst`)

- A single `SyntaxKind` enum covering all token kinds and node kinds (rowan requirement).
- **Lossless lexer**: comments, whitespace, and newlines all become trivia tokens that
  live in the tree (today comments/whitespace are discarded). Significant-newline behavior
  is preserved as a token, exactly as the current parser relies on.
- **Event-emitting parser**: the existing recursive-descent logic is reused but emits
  `start_node` / `token` / `finish_node` into a `GreenNodeBuilder` instead of constructing
  AST structs.
- **Lowering**: `cst → Document` for evaluation. This is the regression net — every
  existing fixture must evaluate to a byte-identical `Value` (equal values hash equal is
  the oracle).

**Sequencing (de-risked):**
- **M9a** — lossless CST + lowering, *keeping bail-on-first-error*. Ships on the
  round-trip + lowering-equivalence nets. Proves the tree shape against the whole language.
- **M9b** — **error recovery**: the parser emits error nodes and resyncs instead of bailing,
  so the LSP gets a tree (and multiple diagnostics) from broken source. Done after M9a so
  the tree shape is already proven.

## Component 2 — Formatter (`mangrove fmt`)

- Walks the CST, re-emits with deterministic layout and comments preserved in place.
- **Canonical layout rules** (pinned so output is stable and idempotent):
  - 2-space indentation.
  - **One binding per line; no trailing commas.** (Lowering treats newline- and
    comma-separated bindings equivalently, so normalizing the separator cannot change
    meaning.)
  - Normalized single-space padding inside `{ }` and `[ ]`; no alignment.
  - Comments stay attached to the construct they lead/trail.
- **CLI:** `mangrove fmt <file>` (rewrite in place), `mangrove fmt --check` (CI; nonzero
  exit if any file would change), `mangrove fmt -` (stdin → stdout; used by the LSP).
- **Two correctness oracles:**
  - *Idempotence*: `fmt(fmt(x)) == fmt(x)`.
  - *Meaning-preservation*:
    - for **evaluable** input — `hash(parse(fmt(src))) == hash(parse(src))`
      (content-addressing proves formatting changed nothing);
    - for **non-evaluable** input (partial/broken/no schema — the cases you most want to
      format) — formatted source re-parses to a CST with **identical non-trivia tokens**
      (structural equivalence).

## Component 3 — LSP server (`mangrove lsp`)

`lsp-server` over stdio. In-memory document store; **full reparse on change**
(*incremental reparse deferred until real files prove large enough to matter*). A
line-index maps offset ↔ UTF-16 `line:character`.

**Committed feature surface ("full"):**
- **Diagnostics** — parse errors (from M9b recovery) + type/compose errors (reuse
  `mangrove-typed` / `-compose`); spans → ranges; published on change. Resolves only
  against the existing lock; never fetches.
- **Hover** — resolved type of the field/symbol under the cursor + its `##` doc comment.
- **Go-to-definition** — **local / same-file** definitions: `use` aliases, local type
  names, params, fn names. (Navigation *into* imported types is stretch — see below.)
- **Completion** — schema field names, type names in type position, keywords.
- **Document symbols** — outline of types / schema / params / fns.
- **Semantic tokens** — classification straight from the CST (types, units, brands,
  strings, interpolation holes, …) → highlighting with no second grammar.
- **Formatting** — `textDocument/formatting` delegates to `mangrove-fmt`.

**Stretch (drops cleanly, not committed):** rename, find-references, and cross-file /
imported-type navigation (the last needs resolver wiring + multi-file document
management).

**Highlighting tradeoff (recorded, accepted):** semantic-tokens means a `.mang` file has
**no highlighting until the LSP attaches**, and none if the server isn't installed. This
keeps one source of truth (the CST) and avoids maintaining a parallel tree-sitter grammar.
A tree-sitter grammar is a noted future option, not part of v0.3.0.

## Component 4 — Editor integration (no VS Code)

- **Neovim** — `editors/nvim/`: `.mang` filetype detection + `nvim-lspconfig` registration
  pointing at `mangrove lsp`. Highlighting via native LSP semantic tokens. Documented setup.
- **Zed** — `editors/zed/`: a small Rust→WASM extension (`zed_extension_api`) registering
  the `Mangrove` language + the LSP command. **Stretch** — drops cleanly if the extension
  API fights us.

## Component 5 — Kubernetes migration docs + example fix

- `docs/migrating-k8s.md`: the `import → add types → export → kubectl apply` loop, a real
  before/after manifest, and the key modeling lesson:
  **model k8s quantities (`512Mi`, `250m`) as branded/refined strings, not unit types** —
  units resolve to a base int (`512Mi` → `536870912`), which k8s accepts but is not
  idiomatic and is non-recoverable (so re-emitting the suffix would break equal-values-
  hash-equal). Branded strings round-trip verbatim. Unit ints remain the right choice when
  you want real arithmetic/refinement on byte counts.
- Fix `examples/k8s-deployment.mang` to use a `Quantity` branded string for memory/cpu so
  it round-trips verbatim.
- **Richer constraint mapping in `gen-openapi`** (so `check` rejects out-of-range / malformed
  values against the real k8s API, not just type/enum mismatches). `enum`→literal union and
  `required` already work; add: `minimum`/`maximum` → `int & >= n & <= m` (or decimal range),
  `pattern` → `str & =~ "…"`, `minLength`/`maxLength` → length refinement. This is the piece
  that makes the LSP's completion/diagnostics catch "bullshit values" live. Independent of the
  CST — only touches `mangrove-openapi` — so it can land ahead of the rest of v0.3.0.

## Testing strategy

- **CST**: lossless round-trip (`tree.to_string() == source` over a corpus — the
  definitive losslessness test); lowering equivalence (every existing fixture evaluates
  identically — reuse the existing parser test corpus as golden); error-recovery fixtures
  (broken input → tree + expected error nodes).
- **Formatter**: idempotence; meaning-preservation (hash-invariance for evaluable inputs,
  structural-equivalence for non-evaluable) over the whole example/fixture corpus; golden
  formatted outputs; comment-attachment cases.
- **LSP**: per-handler tests against marker fixtures (`#^`-style markers naming an expected
  hover type / definition range / completion item / diagnostic at a marked offset); one
  end-to-end stdio test (initialize + a handful of requests). A test asserting the LSP
  performs **no network access** on diagnostics.
- All gated on `just ci` (fmt-check → clippy `-D warnings` → build → test).
